MECHANIC FIELD REFERENCE — what each mechanic needs in JSON
════════════════════════════════════════

Every example object MUST include: "roundNumber" (1–5), "titleText" — this is the MAIN TEXT the child reads at the top of the screen.
MUST be island-themed and narrative, never generic. Write it like a chapter title in an adventure story.

  BAD:  "Solve the problem", "Round 3", "Complete the task", "Math challenge"

Then add ONLY the fields listed for that stage's mechanic. Types: strings unless noted as number or array.

--- drag_sort ---
Game: numbers appear as tokens; child orders them (e.g. ascending).
Per round REQUIRED fields:
  • prompt (string) — comma/space separated values shown as tokens, e.g. "2, 5, 3, 8, 1"
  • answer (string) — MUST be the explicit correct order as a comma-separated sequence matching values from prompt, e.g. "1,2,3,5,8". Do NOT write "ascending" or "descending" — write the actual sorted numbers.

--- drag_group ---
Game: child sorts listed numbers into two named groups (e.g. even / odd).
Per round REQUIRED fields:
  • numbersText (string) — all numbers that appear, e.g. "2, 3, 6, 9, 4, 8"
  • group1Name, group2Name (strings) — labels for the two bins
  • group1Values, group2Values (strings) — comma-separated correct values per group; every number in numbersText must appear in exactly one group

--- pattern_input ---
Game: child types the missing term in a visible sequence.
Per round REQUIRED fields:
  • prompt (string) — sequence with missing slot, e.g. "2, 4, 6, ?"
  • answer (string) — correct missing value

--- fill_blank ---
Game: one equation or phrase with a blank; child types the answer.
Per round REQUIRED fields:
  • prompt (string) — the problem text including the blank, e.g. "12 ÷ 3 ="
  • answer (string) — exact correct value (digits only unless you intentionally use words)

--- multi_choice ---
Game: three tap options A/B/C; exactly one correct.
Per round REQUIRED fields:
  • prompt (string) — task text in the play area
  • choiceA, choiceB, choiceC (strings) — option labels
  • correctOption (string) — exactly one of: "A", "B", "C"

--- corridor_choice ---
Game: child picks left or right path comparing two math expressions.
Per round REQUIRED fields:
  • prompt (string) — short story question, e.g. "Which side is bigger?"
  • leftExpression, rightExpression (strings) — math only, e.g. "3 × 4" vs "2 × 5"
  • correctSide (string) — exactly "left" or "right"

--- match_pairs ---
Game: four statements A–D; exactly two are correct; child finds both.
Per round REQUIRED fields:
  • pairA, pairB, pairC, pairD (strings) — four lines (can be equations with =)
  • correctPair1, correctPair2 (strings) — each is one of "A","B","C","D"; must be two DIFFERENT letters for the two true statements

--- key_lock ---
Game: three lock cards each show a target sum and have two slots; six draggable keys with numbers; child places the correct pair on each lock.
Per round REQUIRED fields:
  • keysSixText (string) — six integers separated by commas, left-to-right order on the key strip, e.g. "4, 11, 15, 6, 9, 0"
  • lock1Sum, lock2Sum, lock3Sum (numbers) — shown on cards; each must equal the sum of that card’s correct pair
  • lock1KeyA, lock1KeyB, lock2KeyA, lock2KeyB, lock3KeyA, lock3KeyB (numbers) — the two keys that belong on that card (order does not matter for the child; must use values from keysSixText)
Optional: prompt — short hint line for that round

--- balance_scale ---
Game: left bowl has an expression with ?; child types what replaces ? to balance the right side.
Per round REQUIRED fields:
  • leftExpression (string) — include "?" for unknown, e.g. "3 × ?"
  • rightExpression (string) — fixed side, e.g. "12"
  • answer (string) — value that replaces ?

--- build_number ---
Game: child builds the base number from part cards; tutor sets base and how many parts.
Per round REQUIRED fields:
  • baseNumber (number) — target whole number
  • partsCount (number) — 2, 3, or 4 (how many part slots)
Optional: readyFlag (boolean) — usually false at generation time

--- timer_challenge ---
Game: three quick problems A/B/C with answers under a time limit.
Per round REQUIRED fields:
  • timerExampleA, timerExampleB, timerExampleC (strings) — problem text
  • timerAnswerA, timerAnswerB, timerAnswerC (strings) — correct answers
  • timerSeconds (number) — seconds allowed, often 30

--- symbol_calc ---
Game: A, B, C shown as values; expression uses letters; child computes final value.
Per round REQUIRED fields:
  • symbolA, symbolB, symbolC (numbers)
  • symbolExpression (string) — uses A, B, C and operators, e.g. "A × B + C"
  • answer (string) — numeric result matching the expression

--- find_unknown ---
Game: A and B known; equation involves C; child solves for C.
Per round REQUIRED fields:
  • unknownA, unknownB (numbers)
  • unknownEquation (string) — must mention C, e.g. "A × B + C = 30"
  • answer (string) — numeric value of C

--- true_false ---
Game: child reads a statement and taps True or False.
Per round REQUIRED fields:
  • statement (string) — the statement shown on the card, e.g. "3 × 4 = 12"
  • trueOrFalse (string) — exactly "true" or "false"

--- text_task ---
Game: child reads a word problem and types the text answer.
Per round REQUIRED fields:
  • prompt (string) — the full task text, e.g. "There are 4 baskets with 3 apples each. How many apples total?"
  • answer (string) — correct answer (checked case-insensitively), e.g. "12"

--- five_tasks ---
Game: five tasks shown simultaneously on one card; all must be answered before advancing.
Per round REQUIRED fields:
  • task1, task2, task3, task4, task5 (strings) — the task text for each row, e.g. "3 × 4 = ?"
  • answer1, answer2, answer3, answer4, answer5 (strings) — correct answer for each row

--- dice_multiply ---
Game: child taps the dice, it rolls 1–6, then multiplies by a set multiplier and types the result.
Rounds 1–3 use multiplier1; rounds 4–5 use multiplier2 (must be harder).
These fields are the SAME for all 5 examples in the stage (only set them in example 1):
  • multiplier1 (number) — multiplier for rounds 1–3, e.g. 3
  • multiplier2 (number) — harder multiplier for rounds 4–5, e.g. 7
  • titleText (string) — e.g. "Roll and multiply!"

--- fortune_wheel ---
Game: child taps Spin, the wheel stops on a random sector (coins or artifact). No math question — this is a reward mechanic.
These fields are the SAME for all 5 examples (only set them in example 1):
  • artifactKey (string) — artifact the wheel can give, e.g. "crown"
  • titleText (string) — e.g. "Spin the wheel!"

--- number_grid ---
Game: a 5×5 grid (rows A–E, columns 1–5) filled with numbers. Child reads a math problem and taps the cell containing the answer.
Example 1 REQUIRED fields (grid is shared across all 5 rounds):
  • gridNumbers (string) — 25 comma-separated numbers to fill the grid, e.g. "34,21,8,15,42,6,18,30,5,27,12,45,3,24,9,36,7,16,25,48,2,20,14,32,10"
Every round REQUIRED fields:
  • prompt (string) — the math problem shown, e.g. "20 + 14 = ?"
  • answer (string) — cell coordinate where the answer appears, row letter + column number, e.g. "B3"
Important: the answer number MUST exist in the grid at the stated coordinate. Design the grid so each round's correct answer is at a different cell.
